# Game Design Document

Let's start from the beginning with the way ULTRAKILL works. Like every game, its gameplay and game design are based on different axioms and sub-axioms, which can and should be understood, used, and combined to create something worthwhile in the context of our mod.

Our work on Items/Perks will focus on (but will not be limited to):

## Lexicon

| Term | Definition |
|---|---|
| **A/R** | Adding/Removing — represents the capacity of certain Perks/Items to increase or decrease certain variables. e.g. A/R ground speed represents the capacity of an item/perk to influence the air speed resulting from a jump after a run or a slide. |
| **INF** | Influence on other variables — certain parts of the character's actions and their environment will be able to apply other effects to other things, in order to try to create original mechanics. It can be almost anything. I will formulate ideas below, in a way that allows effects to be created from actions using the "If" format: if a variable/action exists, and the higher/lower it is, the more it can create a second interesting action elsewhere. e.g. MultiJump → A/R Number of jumps → INF: Chance for mines to appear on the ground at each jump. |
